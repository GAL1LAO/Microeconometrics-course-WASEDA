Do Daily AI Users Earn More Than Non-Users?¶

Naive OLS → Propensity Score Matching (PSM) | Stack Overflow Developer Survey 2025¶

Research question: Does intensive (daily) use of AI coding tools increase developer salary compared to non-users?

  • D (Treatment): AISelect - Daily AI user (1) vs. Non-user (0)
  • Y (Outcome): ConvertedCompYearly - Annual salary in USD (log-transformed)
  • X (Covariates): WorkExp, YearsCode, Age, DevType, Country, OrgSize, EdLevel, RemoteWork, Employment

Workflow (Lec 3-4 methodology):

  1. Load & clean data
  2. Inspect AISelect distribution → motivate treatment definition
  3. Naive OLS (biased baseline)
  4. Logit propensity score model
  5. Nearest-neighbor PSM
  6. Covariate balance check (Love Plot / SMD)
  7. ATT in matched sample
  8. Compare OLS vs. PSM estimates

3. Inspect AISelect Distribution¶

Why we look before defining treatment: In 2025, near-universal AI adoption means a binary Yes/No treatment would violate the overlap condition (P(D=1|X) ≈ 1 for most covariate profiles). We inspect the distribution first to make a data-informed, methodologically sound treatment definition.

Treatment definition (motivated by the data):

  • Treated (D=1): "Yes, I use AI tools daily" - intensive users (~15.8k)
  • Control (D=0): Respondents who answered No or don't plan to use - non-users (~7k)
  • Dropped: Weekly, monthly, NA - intermediate groups create ambiguous counterfactuals

This yields a 2:1 ratio, sufficient for nearest-neighbor PSM without replacement.

4. Select Variables & Clean¶

We select 13 variables from the 172 available columns. The selection follows the DAG logic: only variables that simultaneously affect both salary (Y) and AI tool adoption (D) qualify as confounders (X) and must be controlled for.

Cleaning steps:

  • MainBranch filter: keep only "I am a developer by profession"
  • AISelect filter: keep only Daily users (D=1) and Non-users (D=0). Weekly and Monthly users are dropped because they represent an ambiguous intermediate treatment
  • ConvertedCompYearly: drop missing (~30% of sample, flagged as limitation) and top/bottom 1% outliers
  • log(salary): standard in labour economics (salary is right-skewed)
  • WorkExp / YearsCode: text values ("Less than 1 year", "More than 50 years") converted to numeric (0.5 and 51)
  • Remaining NAs in covariates: listwise deletion

cols = ['ResponseId', 'MainBranch', 'AISelect', 'ConvertedCompYearly',
        'WorkExp', 'YearsCode', 'Age', 'DevType', 'Country',
        'OrgSize', 'EdLevel', 'RemoteWork', 'Employment']
df = df_raw[cols].copy()

# Filter: professional developers only
df = df[df['MainBranch'] == 'I am a developer by profession']

# Filter: keep only Daily users and Non-users (drop Weekly, Monthly, NA)
daily_mask = df['AISelect'] == 'Yes, I use AI tools daily'
no_mask    = df['AISelect'].str.startswith('No', na=False)
df = df[daily_mask | no_mask].copy()

# Binary treatment
df['treat'] = daily_mask[df.index].astype(int)
print("Step 1 - After MainBranch + AISelect filter (before salary & covariate cleaning):")
print(df['treat'].value_counts())
print(f"Subtotal: {len(df):,} rows\n")

# Drop missing salary & extreme outliers (top/bottom 1%)
df = df.dropna(subset=['ConvertedCompYearly'])
low, high = df['ConvertedCompYearly'].quantile([0.01, 0.99])
df = df[(df['ConvertedCompYearly'] >= low) & (df['ConvertedCompYearly'] <= high)]

# Log salary
df['log_salary'] = np.log(df['ConvertedCompYearly'])
print(f"Step 2 - After dropping missing salary & outliers: {len(df):,} rows\n")

# Clean WorkExp & YearsCode
def clean_years(x):
    if pd.isna(x): return np.nan
    if str(x) == 'Less than 1 year': return 0.5
    if str(x) == 'More than 50 years': return 51.0
    try: return float(x)
    except: return np.nan

df['WorkExp']   = df['WorkExp'].apply(clean_years)
df['YearsCode'] = df['YearsCode'].apply(clean_years)

# Drop rows with missing covariates
covariates = ['WorkExp', 'YearsCode', 'Age', 'DevType', 'Country',
              'OrgSize', 'EdLevel', 'RemoteWork', 'Employment']
df = df.dropna(subset=covariates)

6. Naive OLS - Biased Baseline¶

Show the raw association, then argue why it overstates the causal effect (selection bias). Daily users are likely more senior, in better companies, etc.

In [9]:
from statsmodels.iolib.summary2 import summary_col

# Naive OLS
ols_naive = smf.ols('log_salary ~ treat', data=df).fit()

# OLS with controls
ols_controls = smf.ols(
    'log_salary ~ treat + WorkExp + YearsCode + C(Age) + C(DevType) + '
    'C(Country) + C(OrgSize) + C(EdLevel) + C(RemoteWork) + C(Employment)',
    data=df
).fit()

# Koeffizienten
naive_coef = ols_naive.params['treat']
ctrl_coef  = ols_controls.params['treat']
naive_pval = ols_naive.pvalues['treat']
ctrl_pval  = ols_controls.pvalues['treat']
naive_pct  = (np.exp(naive_coef) - 1) * 100
ctrl_pct   = (np.exp(ctrl_coef)  - 1) * 100

# Generate side-by-side regression output table
table = summary_col(
    [ols_naive, ols_controls],
    model_names=['(1) Naive OLS', '(2) OLS + Controls'],
    stars=True,
    info_dict={'N': lambda x: f"{int(x.nobs):,}", 'R²': lambda x: f"{x.rsquared:.3f}"}
)
print(table)

# Calculate magnitude of coefficient shift to evaluate omitted variable bias
bias_magnitude = abs(naive_coef - ctrl_coef)
sign_flip = (naive_coef < 0) != (ctrl_coef < 0)

print(f"\nNaive OLS treatment effect:       {naive_pct:+.1f}%")
print(f"OLS with controls treatment effect: {ctrl_pct:+.1f}%")
print(f"Coefficient change when adding controls: {bias_magnitude:.4f} log points")

if sign_flip:
    print(f"\n→ Sign reversal detected (from {naive_pct:.1f}% to {ctrl_pct:.1f}%)")
    print("→ Strong empirical evidence of selection bias in naive estimate")
    print("→ Confounders (WorkExp, Country, DevType...) were driving the raw difference")
elif bias_magnitude > 0.1:
    print(f"\n→ Large coefficient shift ({bias_magnitude:.3f} log pts) when adding controls")
    print("→ Empirical evidence of omitted variable bias in naive estimate")
else:
    print("\n→ Controls change estimate only marginally - selection bias appears limited")

7. Step 1 of PSM - Logit Propensity Score¶

Estimate P(D=1 | X): probability of being a daily AI user given covariates X. This compresses all covariates into one score for efficient matching (Rosenbaum & Rubin 1983).

# Group rare countries AND DevTypes to prevent perfect separation in logit
def group_rare(series, min_count=30):
    counts = series.value_counts()
    rare = counts[counts < min_count].index
    return series.replace(rare, 'Other')

df['Country_grp'] = group_rare(df['Country'], min_count=30)
df['DevType_grp'] = group_rare(df['DevType'], min_count=30)

8. Step 2 of PSM - Nearest-Neighbor Matching (with Caliper)¶

For each daily user, find the non-user with the closest propensity score. Caliper = 0.1 SD to avoid bad matches.

# Caliper in standard deviations of PS (course convention: 0.25 SD)
ps_sd = df['pscore'].std()
caliper = 0.1 * ps_sd
print(f"PS standard deviation: {ps_sd:.4f}")
print(f"Caliper (0.25 SD): {caliper:.4f}")

# Split into treated and control
treated = df[df['treat'] == 1].copy().reset_index(drop=True)
control = df[df['treat'] == 0].copy().reset_index(drop=True)

# Nearest-neighbor matching WITH replacement
# Replacement is necessary given the 3:1 treated-to-control ratio
# Without replacement, many treated units would receive poor matches
nn = NearestNeighbors(n_neighbors=1)
nn.fit(control[['pscore']])
distances, indices = nn.kneighbors(treated[['pscore']])

treated['match_distance'] = distances.flatten()
treated['match_ctrl_idx'] = indices.flatten()

# Apply caliper - drop treated units without a sufficiently close control match
treated_matched  = treated[treated['match_distance'] <= caliper].copy()
matched_controls = control.iloc[treated_matched['match_ctrl_idx'].values].copy()

# SMD for unweighted data (before matching)
def smd(data, var, treat_col='treat'):
    t = data.loc[data[treat_col]==1, var].dropna()
    c = data.loc[data[treat_col]==0, var].dropna()
    pooled_sd = np.sqrt((t.var() + c.var()) / 2)
    return (t.mean() - c.mean()) / pooled_sd if pooled_sd > 0 else np.nan

# SMD for weighted matched sample (after matching)
def smd_weighted(data, var, treat_col='treat', weight_col='weight'):
    t = data[data[treat_col]==1]
    c = data[data[treat_col]==0]
    mean_t = np.average(t[var].dropna(), weights=t.loc[t[var].notna(), weight_col])
    mean_c = np.average(c[var].dropna(), weights=c.loc[c[var].notna(), weight_col])
    pooled_sd = np.sqrt((t[var].var() + c[var].var()) / 2)
    return (mean_t - mean_c) / pooled_sd if pooled_sd > 0 else np.nan
